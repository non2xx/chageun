# 차근 (chageun)

**비개발자가 안전하게 만들도록 돕는 워크플로우** — 작업카드·검증 게이트·실제 구동 검증·쉬운 말 요약. Claude Code와 Codex CLI 양쪽 지원.

---

## 무엇을 해주나

기획이 약하면 원하던 것과 다른 게 나오고, 검증이 약하면 무한 디버깅 지옥에 갇힙니다. 차근은 **기획 → 게이트 → 구현 → 실제 구동 검증 → 마무리** 흐름으로 그 두 함정을 막아줍니다.

코드는 Claude가, 설명은 쉬운 말로, 판단은 당신이.

---

## 차근의 자리 (2026)

AI 개발의 무게중심은 **프롬프트 → 하네스 → 루프 엔지니어링**으로 옮겨가는 중입니다(Karpathy의 AutoResearch가 대표적 — 프롬프트를 일일이 쓰는 대신 스스로 도는 루프를 설계하자는 흐름). "좋은 지시문 한 줄"에서 "스스로 돌고 검증하는 시스템"으로의 이동입니다.

**카파시는 왜 자신을 루프에서 지웠나.** 그의 관찰은 — 병목은 컴퓨트가 아니라 사람이라는 것입니다. 검증이 자동화·신뢰 가능하고(예: 학습 손실) 되돌리기가 값쌀 때, 사람이 단계마다 결과를 들여다보는 건 정확성을 더하지 못하고 속도만 깎습니다. 그래서 "조건을 세팅하고 물러나는 것"이 최고 레버리지가 됩니다. 단 그조차 "사람을 없앤다"가 아니라 *잡일에서 해방해 창의·판단에 집중*시키는 것입니다.

**그런데 차근은 이 방법론을 절반만 따릅니다.** 카파시의 자율 루프에는 숨은 전제 두 가지가 있습니다 — ① 신뢰할 수 있는 자동 검증 지표 ② 값싼 되돌리기. 비개발자가 만드는 실제 제품엔 둘 다 없습니다: "잘 됐나"를 담을 단일 숫자가 없고, 행동(운영 쓰기·결제·삭제)은 되돌리기 비쌉니다. 게다가 검증을 사람이 떠나면 *의도부채*(왜 이렇게 만들었는지 맥락 유실)가 쌓이고, 비개발자는 자기 분야 밖의 '그럴듯한 가짜'를 걸러내기 어렵습니다.

그래서 차근은 이미 **하네스**이되(매 세션 워크플로우 자동 주입·게이트·검증), 목적함수가 다릅니다:

- **자율 루프**(카파시류): 사람을 루프 밖으로 빼 처리량을 키운다.
- **안전 하네스**(차근): 사람을 *위험 지점에* 남긴다 — 비개발자가 코드를 못 읽어도 안전하게.

차근의 자리는 **"AI는 깊게, 사람은 중요한 데만"**입니다. 위험 없는 작은 일은 가볍게(LIGHT) 흐르고, 위험하거나 애매하면 무겁게(FULL) 가되 — 삭제·배포·운영 쓰기 같은 브레이크는 어떤 경우에도 그대로입니다.

## 설치

### Claude Code

```
/plugin marketplace add chacheum/chageun
/plugin install chageun
```

새 세션이 시작되면 워크플로우가 자동으로 켜집니다.

### Codex CLI

```
codex plugin marketplace add chacheum/chageun
```

그다음 `/plugins`에서 `chageun`를 설치하고 `/reload-plugins`를 실행합니다.

게이트 에이전트를 분리된 컨텍스트로 돌리고 싶으면 `~/.codex/config.toml`에 아래를 추가하세요. 없어도 인라인으로 동작합니다.

```toml
[features]
multi_agent = true
```

## 개발 / 기여

정본(소스)은 `src/`에 있습니다.

```bash
npm run build   # src/ → dist/claude + dist/codex 생성 (결과물은 커밋 대상)
npm test        # 골든·패리티·드리프트 가드 (--test-concurrency=1)
```

`dist/`는 빌드 산출물이지만 **레포에 커밋**합니다 — 마켓플레이스가 소스를 직접 참조하기 때문입니다.

설계·계획 문서는 로컬 `docs/`에 두며 `.gitignore`로 제외됩니다.
